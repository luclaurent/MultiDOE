0001 Toolbox MultiDOE
0002 =======
0003 
0004 This Matlab toolbox regroups many existing tools for generating sample points using many specific DOE techniques:
0005 
0006 * Full factorial sampling
0007 * Latin Hypercube Sampling 
0008 * Optimal Latin Hypercube sampling
0009 * Improved Hypercube Sampling
0010 * Random Sampling
0011 
0012 The toolbox is based on an Oriented-Object syntax for helping to manipulate and generate sample points.
0013 
0014 Many techniques proposed in this toolbox are based on 
0015 
0016 * the [R software](https://cran.r-project.org/). More precisely, the library [lhs](https://cran.r-project.org/web/packages/lhs/index.html) is used. 
0017 * the [IHS](https://people.sc.fsu.edu/~jburkardt/m_src/ihs/ihs.html) matlab's software proposed by [John Burkardt](http://people.sc.fsu.edu/~jburkardt/).
0018 * the [LHS](http://www.mathworks.com/matlabcentral/fileexchange/4352-latin-hypercube-sampling/content/lhsu.m) proposed by Budiman Minasny.
0019 
0020 **NOTICE: sampling techniques based on the R software are only usable on a Unix OS.**
0021 
0022 First start
0023 ------
0024 
0025 Some examples are proposed by functions:
0026 
0027 * `ExampleUse.m` simple example of the use of the toolbox
0028 * `ExampleInfill.m` example of the use of the toolbox with infill of the initial sampling

0031 Features
0032 ------
0033 The MultiDOE toolbox are able to 
0034 
0035 * generate sampling
0036 * sort the sample points using many techniques
0037 * add points to an existing sampling (based on the LHS and IHS procedures of the R software)
0038 * normalize/renormalize the sample points to be well defined in the specified design space
0039 * call automatically the R software (on Unix/Linux/OSX systems only)
0040 * compute scores (minimal distances, discrepancy...), see for instance the [Franco's PhD thesis (2008)](https://tel.archives-ouvertes.fr/tel-00803107/)
