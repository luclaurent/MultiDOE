SOURCE CODE 


0001 Toolbox MultiDOE
0002 =======
0003 
0004 This Matlab toolbox regroups many existing tools for generating sample points using many specific DOE techniques:
0005 
0006 * Full factorial sampling
0007 * Latin Hypercube Sampling 
0008 * Optimal Latin Hypercube sampling
0009 * Improved Hypercube Sampling
0010 * Random Sampling
0011 
0012 Many techniques propose in this toolbox are based on 
0013 
0014 * the [R software](https://cran.r-project.org/). More precisely, the library [lhs](https://cran.r-project.org/web/packages/lhs/index.html) is used. 
0015 * the [IHS](https://people.sc.fsu.edu/~jburkardt/m_src/ihs/ihs.html) matlab's softwares proposed by [John Burkardt](http://people.sc.fsu.edu/~jburkardt/)
0016 * the [LHS](http://www.mathworks.com/matlabcentral/fileexchange/4352-latin-hypercube-sampling/content/lhsu.m) proposed by Budiman Minasny
0017 
0018 Features
0019 ------
0020 The MultiDOE toolbox are able to 
0021 
0022 * generate sampling
0023 * sort the sample points using many techniques
0024 * add points to an existing sampling (based on the LHS and IHS procedures of the R software)
0025 * normalize/renormalize the sample points to be weel defined in the specified design space
0026 * call automatically the R software (on Unix/Linux/OSX systems only)
0027 * compute scores (minimal distances, discrepancy...), see for instance the [Franco's PhD thesis (2008)](https://tel.archives-ouvertes.fr/tel-00803107/)
0028 
0029 
0030 
0031 Requirements
0032 ------
0033 The toolbox requires:
0034 
0035 * the [Statistics and Machine Learning Toolbox](http://fr.mathworks.com/products/statistics/) of the Matlab's software
0036 * the R software installed on an UNIX/Linux/OSX system and the `R` command available on the command line
0037 * the [lhs](https://cran.r-project.org/web/packages/lhs/index.html) and [R.matlab](https://cran.r-project.org/web/packages/R.matlab/index.html) installed on R (see for instance [How to Install an R Package](http://math.usask.ca/~longhai/software/installrpkg.html))
0038 * the [graphviz software](www.graphviz.org) (the `dot` command must be available on the command line) for building the documentation (optional).
